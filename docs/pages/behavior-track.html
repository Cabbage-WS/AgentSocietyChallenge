User Modeling Track

Introduction

In this track, participants will use a classic open-source dataset to simulate social interactions and information access behaviors on social media. This is crucial for designing effective information retrieval algorithms for web applications. Using LLM-based agents, participants will combine historical user behavior records and interact with the environment to model how users might review businesses and engage with reviews.

The track focuses on User Preference Simulation: Participants should design LLM agents that can understand user preferences and behavior patterns to model user preferences. The agent may integrate user characteristics, business attributes, and other users' reviews to form a comprehensive judgment. Each review must include two components:

  • Star rating: A numerical score reflecting the user's opinion.
  • Review text: Detailed commentary providing specific feedback.

This track explores the potential and scalability of LLM agents in simulating human behavior, offering insights into their applications in behavioral modeling.

Evaluation

In the User Behavior Modeling Track, participants are expected to use LLM agents to model user interactions with recommended items (e.g., businesses on Yelp), including star ratings and review texts. The evaluation for this track will use quantitative metrics to assess the accuracy of the models in simulating user behavior.

Specifically, the predicted star ratings are compared with the ground truth using Mean Absolute Error (MAE) after normalizing the values to the range [0,1]. The MAE is calculated as:

MAE = (1 / N) ∑ | ŝni - sni |

Where N is the total number of reviews, and ŝni and sni are the i-th normalized predicted star and normalized ground truth, respectively.

For the generated review texts, semantic similarity will be evaluated across predefined sentiment dimensions, such as emotional tone, sentiment attitude, and topic relevance.

  • Emotional tone and sentiment attitude: Calculated using predefined models, normalized to [0,1], and computed through MAE.
  • Topic relevance: Measured using cosine distance between text embeddings, defined as:

d = 1 - (u · v / ||u||2 · ||v||2)

Finally, after obtaining the individual results, the final error score will be a weighted sum of these metrics.
